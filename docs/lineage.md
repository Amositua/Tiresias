# Lineage: Blast-Radius Tracing

## How it works

`LineageGraph` parses a dbt `manifest.json` and builds a directed dependency graph
(upstream → downstream) using networkx. Given a changed source table and optionally
a changed column, `blast_radius()` does a BFS from that source node and returns every
downstream model, view, and exposure ordered nearest-first, with severity assigned by
hop distance:

| Hop | Severity |
|-----|----------|
| 1   | critical |
| 2   | high     |
| 3   | medium   |
| 4+  | low      |

For each downstream node, the engine checks whether the changed column name appears
as a substring in the node's SQL (`raw_code` / `compiled_code`). Nodes that directly
reference the column are flagged `references_column = True`. Exposure nodes carry the
owner name or email from the manifest.

## The manifest fixture

`backend/tests/fixtures/manifest.json` is a structurally valid dbt v9 manifest
constructed to match the demo scenario. It is not generated by dbt and does not
reflect a live project — it exists so that `LineageGraph` can be tested and
demonstrated without access to the customer's actual dbt project.

In production, `LineageGraph` reads the customer's real `manifest.json` unchanged —
the one produced by `dbt compile` or `dbt docs generate`. The fixture and the
production manifest are byte-for-byte identical in schema; only the content differs.

## Demo scenario trace

Starting from `hubspot.deal_pipeline_stage`, changed column `label`:

```
Blast radius for deal_pipeline_stage.label:
  [CRITICAL] model: stg_deals [references column]
  [HIGH] model: fct_pipeline_by_stage
  [MEDIUM] exposure: late_stage_pipeline_dashboard (owner: VP of Sales)
```

`stg_deals` is flagged `[references column]` because its SQL contains the literal
`s.label = 'Contract Sent'` — the exact predicate that silently returns $0 after the
HubSpot stage rename.

## API

```python
from lineage.graph import LineageGraph

g = LineageGraph("/path/to/manifest.json")
g.load()

br = g.blast_radius("deal_pipeline_stage", "label")
print(br.as_summary())

# Access structured data
for node in br.nodes:
    print(node.node_type, node.name, node.severity, node.references_column)
```

`blast_radius()` accepts `"table_name"` or `"source_name.table_name"`.
If the source is not found, it returns a `BlastRadius` with an empty `nodes` list.
